### `tools/validate_art_assets.gd`

`@tool extends EditorScript` — report-only preflight. Run via Tools → Execute Script.

**Methods:**

| Method | Purpose |
|---|---|
| `_run()` | Entry point; clears reports, triggers scan, prints report |
| `_scan_directory(dir_path: String)` | Recursively walks `res://art/` via `DirAccess` |
| `_infer_category(glb_path: String) -> String` | Maps path prefix to category string |
| `_get_required_clips(category: String) -> PackedStringArray` | Returns required clip names per category |
| `_check_glb(report: GlbReport)` | Loads GLB via `ResourceLoader.load()`, instantiates scene, checks clips |
| `_find_animation_player(root: Node) -> AnimationPlayer` | Depth-first search for AnimationPlayer in subtree |
| `_report()` | Prints per-file results and summary to Godot output panel |

**Required clips per category:**

| Category | Required Clips |
|---|---|
| `enemy` | idle, walk, death, hit_react |
| `ally` | idle, walk, death, attack_melee |
| `building` | idle, active |
| `boss` | idle, walk, death, phase_transition |
| `tower` | idle |
| `unknown` | (none) |

**Notes:**
- Uses `ResourceLoader.load()` — avoids `GLTFDocument.generate_scene()` silent track drop gotcha.
- Uses `DirAccess` recursive scan — does not depend on `EditorFileSystem`.
- Report-only: no assets are created or modified.
