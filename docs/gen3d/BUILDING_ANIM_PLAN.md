# Building Animation Plan

**Last updated:** 2026-04-22

---

## Overview

Buildings use three animation clips: `idle`, `active`, and `destroyed`.
Their authoring approach differs from humanoid characters because buildings have
no skeleton and do not use Mixamo FBX files.

---

## 1. `idle` â€” Static Single-Frame Action

**Authoring:** Synthesised by Stage 4 in Blender automatically â€” no `Idle.fbx` needed.

Stage 4 should detect `asset_type == "building"` and inject a minimal Blender script
that inserts a single keyframe at frame 0 for every object's transforms, then exports
as an `AnimationPlayer` action named `"idle"`.

**Godot behaviour:** `AnimationPlayer.play("idle")` loops the single frame â€” effectively
a static pose. This is intentional; the building simply sits still.

**Required Stage 4 change:**  
In `stage4_anim.py`, add a branch for `asset_type == "building"`:

```python
if asset_type == "building":
    _merge_building_animations(rigged_glb, out_path)
    return str(out_path)
```

`_merge_building_animations()` uses a Blender script that:
1. Imports the building GLB
2. Inserts a single keyframe at frame 0 (location/rotation/scale) for each mesh object
3. Names the action `"idle"`
4. Conditionally imports `anim_library/buildings/BuildingActive.fbx` if present â†’ names it `"active"`
5. Exports GLB with all actions

---

## 2. `active` â€” 2-Bone Rotation (Manual Blender Authoring)

**Authoring:** Manual â€” not generated by the pipeline.

A Blender artist creates a short looping animation (e.g. a turret barrel rotating,
a fan spinning, a cauldron bubbling) using simple bone or object keyframes. This is
exported as:

```
tools/gen3d/anim_library/buildings/BuildingActive.fbx
```

The FBX should contain a single action at 30 fps. The clip length should be a clean loop
(e.g. 30 frames = 1 second at 30 fps).

**Stage 4 consumption:** `_merge_building_animations()` checks for
`anim_library/buildings/BuildingActive.fbx` and imports it if found. If absent, the
`active` clip is omitted silently (building still works with only `idle`).

**Godot usage:**
```gdscript
if animation_player.has_animation(RiggedVisualWiring.ANIM_ACTIVE):
    animation_player.play(RiggedVisualWiring.ANIM_ACTIVE)
```

## 3. `destroyed` â€” Blender Shape Key (Morph Target)

**Authoring:** Manual â€” shape key baked in Blender, not a skeletal animation.

The building mesh is exported from Blender with a shape key named `"collapsed"` or
`"key_0"` (value 0 = intact, 1 = destroyed/collapsed geometry).

**GLB export requirement:** Blender â†’ File â†’ Export glTF 2.0 with:
- `Include â†’ Shape Keys` enabled
- `Include â†’ Animations â†’ Shape Key Animations` enabled (optional, if driving via NLA)

**Godot `AnimationPlayer` track:** In the Godot scene or via a script, an `AnimationPlayer`
drives the morph target from 0 â†’ 1 over 0.8 seconds on destruction:

```
Track path: "MeshInstance3D:blend_shapes/key_0"   (or the exact shape key name)
  Key 0.0s  â†’  0.0   (intact)
  Key 0.8s  â†’  1.0   (fully destroyed/collapsed)
```

This produces a smooth geometry morph without requiring a skeleton.

**Godot usage:**
```gdscript
# In building_base.gd or similar â€” called when building HP reaches 0:
func _play_destroyed() -> void:
    if animation_player != null and animation_player.has_animation(RiggedVisualWiring.ANIM_DESTROYED):
        animation_player.play(RiggedVisualWiring.ANIM_DESTROYED)
```

## Required Changes to `ANIM_CLIPS_BUILDING`

Current (in `foulward_gen.py`):
```python
ANIM_CLIPS_BUILDING: list[str] = ["idle", "active", "destroyed"]
```
No change required to the clip list itself.

Stage 4 must route `asset_type == "building"` through `_merge_building_animations()`
instead of the humanoid Mixamo FBX path. The `destroyed` clip does not come from an
FBX file â€” it is a shape key baked into the mesh at authoring time. Stage 4 should
not attempt to import `Falling Back Death.fbx` for buildings.
